# Schengen Visa Skill — 设计文档

Version: v1 | Date: 2026-05-19 | Target: 在校学生（中国护照）

## 概述

一个交互式申根签证向导 skill。面向持中国护照的在校大学生，引导完成从选申请国到递签的全流程。聚焦法国/意大利/西班牙三个高通过率国家，其他国家走自适应流程。首版只做学生，后续扩展在职/自由职业。

## 设计目标

- 用户说"我要办申根签"即触发
- 交互式引导，一次只问一个问题
- 每一步完成后更新落盘进度文件
- 最终确认环节逐条检查 15 项材料，防止遗漏
- 本质目的是下签，所有建议围绕过签率优化

## 交互流程（5 步）

### 第一步：选定身份 + 申请国

1. 确认用户是在校学生（硬编码，首版不做其他身份）
2. 推荐 3 个高通过率国家：
   - 法国：出签快、不分领区、全国可递，但给的天数偏紧（按行程单来）
   - 意大利：通过率不错
   - 西班牙：有时会比行程多给几天
3. 用户可选这三个之一，也可自选其他国家
4. 选定后必须提醒：行程单里该国天数写最长，出入境机票最好也是该国，材料上别穿帮

### 第二步：网上预约（国家自适应）

不同国家签证外包商不同，Claude 自行查询当前国家的签证中心官网：
- 法国 → TLScontact
- 意大利 → VFS Global
- 西班牙 → BLS

引导完成：
1. 官网填申请表 → 生成申请号（截图保存）
2. 注册签证中心 → 关联申请号 → 选城市预约
3. 打印申请表 + 预约单（计入 15 项中的 2 项）

### 材料准备的正确顺序

三者必须串成一条线，互相吻合：

1. **先做行程单**（哪天在哪个城市）
2. **再订酒店**（跟着行程单上的城市和日期匹配）
3. **最后订机票**（出入境城市和日期呼应行程单）

→ 三者都指向申请国，签证官看不出破绽。

其中酒店预订的分工：
- Claude 按行程单列出需求清单（X 月 X 日-X 月 X 日，住 XX 城市）
- 用户自己去携程/Booking 选免费取消的下单、下载确认单
- Claude 帮检查确认单信息是否与行程单一致

### 第三步：材料准备（15 项）

按以下分类逐项引导，输出到 progress.md：

**证件类（4 项）**
1. 护照原件 + 所有面复印件（不要空白页）
2. 身份证原件 + 复印件
3. 户口本整本复印件
4. 两寸白底证件照（现场 35 块拍也行）

**学生身份类（3 项）**
5. 中英文在读证明（老师签字/学校公章）
6. 学生证原件 + 复印件
7. 学信网中英文学籍证明

**资金类（1 + 备用方案）**
8. 最近三个月银行流水（余额 ≥ 3 万）
   - 父母出资：需父母流水 + 父母在职证明；不在同一户口本需亲属关系公证
   - 个人出资：解释信说明资金来源

**旅行保障（1 项）**
9. 保险：京东/支付宝买，彩打前两页，日期完整覆盖行程，可退

**行程与机酒（3 项）**
10. 行程单：
    - 推荐「行程助手」app 自行制作
    - 或让 Claude 直接生成送签标准行程单
    - 要求：申请国天数最长、一天 3-4 景点、城市间移动合理、景点营业时间检查、不要特种兵
    - 本质是送签文件，不是真实旅行计划
11. 机票预订单：trip.com 预订不付款即可打印
12. 酒店预订单：携程免费取消/先用后付

**灵魂材料（1 项）**
13. 解释信（见 references/cover-letter.md）：
    - 我们是跟签证官直接对话的唯一机会
    - 解释资金疑点（临时转账、余额来源等）
    - 强调回国约束力（在读学生、学业未完成、未来规划、家人在国内等）
    - 让签证官觉得"有钱消费 + 一定会回国"

**预约凭证（2 项）**
14. 签证申请表（第一步官网填的）
15. 签证中心预约单

### 第四步：递签当天

- 按时到签证中心
- 所有材料不要用订书机或夹子装订
- 材料按顺序整理好
- 可能有电调：如实回答，别紧张

### 第五步：等结果

- 邮寄约一周（上海领区较快）
- 建议留半个月缓冲
- 可选：出签后退保险

## 进度追踪机制

**敏感材料（证件类、身份类、资金类）：只提醒，不生成，不接触内容**
- Claude 告知用户需要准备什么、去哪办
- 用户说"XX 好了"即标记完成，Claude 不追问具体信息
- 不要求用户上传或粘贴敏感文件内容

**非敏感材料（行程单、解释信、保险等）：可帮生成或检查**

**进度落盘**：
- 15 项 checkbox 写入 `progress.md`
- 用户说"XX 好了" → Claude 立即勾选对应项并落盘
- 每轮对话末尾报缺："你还差 X 项，下一步建议 Y"
- 优先催有依赖关系的（如：在读证明要等老师签字）

## 最终确认

用户说"准备好了"时触发，逐条复盘——包括用户之前已标记完成的项也要再次确认：
- 15 项是否全部完成？（逐项过）
- 证件/身份/资金类材料再次确认是否已拿到手
- 保险日期完整覆盖行程？
- 流水是最近三个月？
- 在读证明有签字/公章？
- 机票/酒店订单名称与护照一致？
- 行程单国家天数与申请国匹配？
- 所有材料无订书机/夹子？
- 材料顺序正确？

全部通过 → "可以去递签了，祝顺利出签。"

## 解释信设计要点

见 references/cover-letter.md，核心策略：
- 学生身份优势大——回国约束力天然强
- 资金疑点必须解释：临时大额转账来源、平时钱放哪
- 长行程要合理化：寒假/暑假有的是时间
- 强调"第一次去欧洲所以想多看看"
- 可让 ChatGPT/Claude 代笔，用户审核修改

## 各国差异

见 references/country-notes.md：
- 法国：签证中心 TLScontact，不分领区
- 意大利：VFS Global
- 西班牙：BLS
- 其他国家：Claude 自行查询
- 通用原则：行程单配合申请国
