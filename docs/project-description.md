# Schengen Visa Skill — 项目说明

申根签证是中国护照持有者最常见的出境签证之一，DIY 申请需要准备十五项材料，缺一不可。市面上的攻略文章很多，但攻略是死的，它不能根据你的身份告诉你该走学生出资还是父母出资，也不能在你第二天打开电脑时记得你昨天办到哪了。这个 skill 做的事情，就是把一条死攻略变成一个能跨会话连续工作的交互式向导。

它解决的第一个痛点是流程碎片化。申根签 DIY 最难的不是填表，而是不知道该从哪一步开始，先订机票还是先做行程单，保险什么时候买，在读证明开了会不会过期。skill 把全流程固化为一条线性的步骤链，每一步完成之后自然推到下一步。

第二个痛点是跨会话记忆断裂。办签证通常要好几天，聊半小时办完了五项材料，关掉窗口，第二天 AI 连你在办签证都不知道。这个 skill 的核心改进就是引入了一个 progress 文件作为落盘机制，每次用户说某项材料好了，模型立刻写入文件勾掉对应的复选框。每次新会话启动，模型被强制要求先读取进度文件，确认上次办到哪了再从断点继续。用户不用复述，AI 先说。

第三个痛点是敏感信息与可生成内容之间没有边界。证件照、户口本、银行流水、在读证明属于敏感个人信息，AI 不应该接触，但行程单、解释信又完全可以帮忙生成。skill 预设了这条分界，敏感材料只提醒怎么办，办完了勾掉即可，非敏感材料可直接代劳。

第四个痛点是申根二十九国各有一套流程，签证外包商不同，官网不同，领区划分不同。skill 内置了全部二十九国的官方直链，法意西三国还有详细步骤，用户选哪个国家就从 reference 文件里查哪个国家的链接和流程，不用自己搜。

架构上这是一个纯 markdown 的技能系统，没有代码和数据库。主控文件 SKILL.md 约一百六十行，内含完整交互流程和状态管理规则。四个 reference 文件各自负责材料清单、各国网址和流程、解释信模板、行程单指南，只在对应步骤被引用，不会全部常驻上下文。

核心逻辑可拆为五个机制。一是会话启动门，每次新会话模型先读 progress 文件再回复用户，这是写死的强制指令。二是逐项推送加即时落盘，五步主线走完，材料阶段按证件、学生身份、保险、资金、行程单、机票、酒店、解释信的依赖链排序，用户说好了模型先写文件勾掉再回复，先存再回。三是申请国自适应路由，选国后从 country-notes 查直链和流程，这是查表不是推理。四是敏感与非敏感边界，证件学籍资金只提醒，行程单解释信可生成。五是最终确认，不是看十五项全绿就完事，而是逐项再过一遍，连已勾掉的也要重新确认，因为勾选和材料真正就位之间可能有差距。

关于多 Agent 协作和长链推理，当前版本都没有。这是一个单人机交互式向导，智能不是来自模型自主规划，而是来自流程设计中的依赖关系预设、边界条件的硬约束和状态的落盘恢复。思路是把智慧写进文档结构里，让模型按文档执行，而不是让模型自己想怎么办。

目前仅支持中国护照加在校大学生，仅覆盖申根短期旅游签。其他身份和签证类型尚未覆盖，非申根国家会主动拒绝。
